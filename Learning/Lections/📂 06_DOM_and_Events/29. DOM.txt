Тема №29: DOM. Пошук елементів (DOM. Selecting Elements) 🌐
🌐 Що таке DOM?
DOM (Document Object Model) — це перетворення HTML-сторінки на дерево об'єктів, яким може керувати JavaScript. 
Коли браузер завантажує твій HTML-файл, він створює в пам'яті копію кожного тегу у вигляді об'єкта. 
JS може ці об'єкти шукати, видаляти, змінювати їм кольори чи текст.

🔌 Як пов'язати HTML та JavaScript?
Щоб твій JS-код почав керувати сторінкою, його потрібно підключити всередині HTML-файлу. Для цього використовується тег <script>.
Найкраще підключати його в самому кінці тегу <body>, щоб спочатку завантажилися всі візуальні елементи, а вже потім виконувався код.
Приклад підключення: <script src="app.js"></script> (де app.js — назва твого JS-файлу).

🎯 Звідки беруться селектори для пошуку?
JavaScript не вигадує назви елементів. Він бере їх прямо з HTML. Для пошуку використовуються CSS-селектори. 
Це загальноприйнята система в усьому вебі:
🏷️ Пошук за тегом: Якщо шукаємо стандартний тег (наприклад, кнопку <button> чи абзац <p>), у JS ми пишемо просто ім'я тегу: 'button' або 'p'.
🔴 Пошук за класом: Класи розробник придумує сам для стилів чи логіки (наприклад, class="main-btn"). 
Щоб JS зрозумів, що це клас, перед назвою обов'язково ставиться крапка: '.main-btn'.
🆔 Пошук за ID: Унікальний ідентифікатор елемента (наприклад, id="page-title"). 
Щоб JS зрозумів, що це ID, перед назвою обов'язково ставиться решітка: '#page-title'.

🔍 Сучасні методи пошуку в JavaScript:
Всі пошуки починаються з глобального об'єкта document, який представляє всю твою сторінку.
document.querySelector('селектор') — шукає і повертає один найперший елемент на сторінці, який підійшов під умови пошуку.
document.querySelectorAll('селектор') — шукає і повертає всі елементи на сторінці у вигляді колекції (подібної до масиву).

Тема №2: Запуск JS у браузері та консоль розробника (Running JS in Browser and DevTools) 🌐
Браузер (Google Chrome, Microsoft Edge тощо) — це природне середовище для JavaScript. 
Саме всередині браузера існує той самий об'єкт document, який ми намагаємося викликати.

Ось, як виглядає звичайний html код, що знаходиться у файлі index.html який тобі треба буде створити і з яким ми будемо працювати 
У VS Code у тій же папці, де лежить твій app.js, створи новий файл з назвою index.html та встав у нього цей код:

HTML

<!DOCTYPE html>
<html lang="uk">
<head>
    <meta charset="UTF-8">
    <title>Вивчення DOM</title>
</head>
<body>

    <h1 id="main-title">Вітаємо у світі DOM!</h1>
    <p class="text-info">Це перший абзац з важливою інформацією.</p>
    <p class="text-info">Це другий абзац для кількості.</p>
    
    <button class="action-btn">Натисни мене</button>
    <button class="action-btn">Або мене</button>

    <script src="app.js"></script>
</body>
</html>

✏️ Що таке .textContent?
Це властивість об'єкта елемента, яка містить у собі весь текст, що знаходиться всередині HTML-тегу. 
Ми можемо як читати цей текст, так і перезаписувати його.

🔹 Як це працює на практиці?
Спочатку ми знаходимо елемент і зберігаємо його у змінну.
Через крапку звертаємося до властивості .textContent.
За допомогою знака дорівнює = присвоюємо новий рядок.
// 1. Знаходимо заголовок
const mainHeader = document.querySelector('#main-title');

// 2. Змінюємо його текст на новий
mainHeader.textContent = "Привіт з JavaScript!"; 

Щойно браузер виконає цей рядок коду, старий текст «Вітаємо у світі DOM!» миттєво зникне, 
а замість нього користувач побачить «Привіт з JavaScript!».

🔹 Зміна тексту в колекціях (querySelectorAll):
Якщо ти знайшов кілька елементів через querySelectorAll, ти не можеш змінити текст усім одразу ось так: allButtons.textContent = "Текст". 
Це викличе помилку, бо allButtons — це колекція (масив).
Щоб змінити текст кожній кнопці, масив потрібно перебрати через вже знайомий тобі метод .forEach():

const allButtons = document.querySelectorAll('.action-btn');
allButtons.forEach((btn, index) => {
    btn.textContent = `Кнопка №${index + 1}`;
});

🖌 Властивість .style
Кожен DOM-елемент має об'єкт .style. Записуючи туди значення, ми ніби додаємо інлайнові стилі безпосередньо в HTML-тег 
(наприклад, <p style="color: red;">).

🔹 Важливе правило написання назв властивостей (CamelCase):
У CSS ми пишемо властивості через дефіс: background-color, font-size.
Але в JavaScript дефіс означає мінус! Тому всі назви властивостей в JS пишуться за правилом camelCase 
(без дефісів, а кожне наступне слово з великої літери):

background-color ➡️ backgroundColor
font-size ➡️ fontSize
text-align ➡️ textAlign

🔹 Приклад роботи зі стилями:
const mainTitle = document.querySelector('#main-title');

if (mainTitle) {
    mainTitle.style.color = 'red';               // Змінюємо колір тексту
    mainTitle.style.backgroundColor = 'yellow';  // Змінюємо колір фону
    mainTitle.style.fontSize = '40px';           // Змінюємо розмір (обов'язково вказувати одиниці виміру 'px' як рядок)
}

👁️ Як ховати та показувати елементи на сторінці?
У CSS є чудова властивість display. Якщо ми дамо їй значення 'none', елемент повністю зникне зі сторінки, 
ніби його там ніколи й не було. А якщо ми хочемо його повернути, ми повертаємо йому стандартне значення 
(наприклад, '' — порожній рядок, що скидає приховування, або 'block').
element.style.display = 'none'; // Елемент зникає з екрана 👻
element.style.display = ''; // Елемент знову з'являється 🔑

🧱 Робота з декількома елементами одночасно
Коли ми пишемо код для реальних завдань, ми спочатку знаходимо всі потрібні нам «вузли» (елементи) сторінки, 
перевіряємо їх за допомогою нашого правила if, а потім крок за кроком міняємо їм і текст, і стилі.
Наприклад, якщо ми робимо "Нічний режим" для сайту, ми маємо знайти фон сторінки, знайти текст, 
знайти кнопки і кожному з них прописати нові кольори.
