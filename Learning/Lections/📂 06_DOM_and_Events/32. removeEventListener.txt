Тема №32: Видалення обробників подій (removeEventListener) 🗑️
🗑️ Навіщо видаляти слухачі подій?
Коли ми додаємо слухач подій за допомогою .addEventListener(), браузер виділяє під нього оперативну пам'ять. 
Якщо елемент видаляється зі сторінки або подія більше не потрібна за логікою програми, але слухач залишається «висіти», 
це призводить до неприємних наслідків:
🛑 Витік пам'яті (Memory Leaks): Додаток починає споживати все більше ресурсів комп'ютера, що з часом уповільнює роботу вкладки.
🪲 Логічні баги: Код продовжує реагувати на дії користувача тоді, коли це вже заборонено 
(наприклад, гра закінчилася, з'явилося вікно "Game Over", але кліки по фону сторінки досі зараховують ігрові бали).

⚠️ Головна пастка .removeEventListener()
Щоб видалити слухача, метод .removeEventListener() повинен отримати в аргументи точно таку саму функцію, яка передавалася під час його створення.

Через це ми не можемо видалити анонімну функцію:
// Помилка: цей рядок нічого не видалить!
button.addEventListener('click', () => console.log('Клік'));
// Браузер створить у пам'яті НОВУ анонімну функцію, яка просто схожа на першу
button.removeEventListener('click', () => console.log('Клік'));

💡 Аналогія з C#: Це працює точно так само, як відписка від подій через оператор -=. 
Ти не можеш відписати анонімний лямбда-вираз, якщо не зберіг посилання на нього в делегат.

🛠️ Правильний підхід: Іменовані функції
Щоб видалення спрацювало бездоганно, функцію потрібно створити окремо, дати їй ім'я і передавати це ім'я як посилання (без дужок ()):
// 1. Створюємо чітку іменовану функцію
function handleButtonClick() {
    console.log("Кнопку натиснуто!");
}
// 2. Додаємо її як обробник
button.addEventListener('click', handleButtonClick);
// 3. Видаляємо її, коли потреба зникла
button.removeEventListener('click', handleButtonClick);

📌 Правило однієї дії:
Якщо твоя кнопка чи елемент створені для однієї конкретної одноразової дії (наприклад, кнопка «Підтвердити замовлення», 
яка після кліку має заблокуватися, або картка, яка має зникнути), — ми використовуємо .removeEventListener().
Якщо це звичайна кнопка, яка має працювати завжди (наприклад, кнопка «Плюс» у калькуляторі, яка постійно додає числа), — 
ми .removeEventListener() не пишемо. Вона собі спокійно живе на сторінці.

🔒 Керування станом елементів (disabled)
Коли ми хочемо не просто відв'язати функцію, а й візуально показати користувачеві, що кнопка більше недоступна, 
ми можемо заблокувати її за допомогою властивості .disabled = true;.

📝 Завдання №32.1 (Одноразова кнопка):
Давай закріпимо цю тему на практиці. Ми створимо кнопку, яка після першого ж кліку сама себе "деактивує" шляхом видалення слухача.
Крок 1. Повністю заміни код у своєму index.html на цей мінімалістичний варіант:
<!DOCTYPE html>
<html lang="uk">
<head>
    <meta charset="UTF-8">
    <title>Видалення подій</title>
    <style>
        body { font-family: sans-serif; padding: 50px; text-align: center; }
        #trigger-btn { padding: 10px 20px; font-size: 16px; cursor: pointer; }
    </style>
</head>
<body>

    <button id="trigger-btn">Натисни мене (Тільки 1 раз)</button>

    <script src="app.js"></script>
</body>
</html>

Крок 2. Повністю очисть файл app.js і напиши код за такими підказками (крок за кроком, як ти любиш):
Знайди кнопку за її ID #trigger-btn. Обов'язково додай коментар типу / @type {HTMLElement} */ та перевірку if.
Вище або всередині блоку if оголоси класичну іменовану функцію з назвою onButtonClick.
Всередині функції onButtonClick реалізуй дві дії:
Зміни текст кнопки на "Мене можна натиснути лише раз!" за допомогою ключового слова this.textContent.
Одразу наступним рядком повністю видали цей обробник з кнопки за допомогою this.removeEventListener('click', onButtonClick);.
В самому кінці (всередині блоку if) прив'яжи функцію до кнопки через addEventListener на подію 'click'.

✅ Рішення:
/** @type {HTMLElement} */
const button = document.querySelector('#trigger-btn');
if(button){
    function onButtonClick(){
        this.textContent = 'Мене можна натиснути лише раз!';
        this.removeEventListener('click', onButtonClick);
    }
    button.addEventListener('click', onButtonClick);
}

📝 Завдання №32.2 (Панель безпеки):
Давай створимо систему з двох кнопок. Перша кнопка буде виводити повідомлення в консоль, а друга — виступатиме в ролі «запобіжника», 
який цю першу кнопку повністю знешкоджує.

Крок 1. Онови свій index.html, щоб у тілі сторінки були дві кнопки:
<button id="action-btn">Виконати важливу дію</button>
<button id="deactivate-btn">Ввімкнути захист (Знешкодити першу кнопку)</button>

Крок 2. У файлі app.js реалізуй таку логіку:
Знайди обидві кнопки за їхніми відповідними ID (не забудь про JSDoc / @type {HTMLElement} */ та перевірку if для обох елементів).
Оголоси окрему іменовану функцію з назвою executeAction. Всередині неї напиши лише одну дію:
Виводь у консоль повідомлення: "Важливу дію виконано успішно!".
Оголоси другу іменовану функцію з назвою disableAction. Всередині неї реалізуй такі кроки (працюй через змінні кнопок або this):
Зміни текст другої кнопки (самої себе) на: "Захист активовано!".
Зміни текст першої кнопки на: "Дія заблокована".
Головне: повністю видали обробник executeAction з першої кнопки.
В самому кінці (всередині перевірки if) прив'яжи відповідні функції до подій 'click' для кожної кнопки.

✅ Рішення:
/** @type {HTMLElement} */
const action = document.querySelector('#action-btn');

/** @type {HTMLElement} */
const deactivate = document.querySelector('#deactivate-btn');

if(action && deactivate){
    function executeAction(){
        console.log("Важливу дію виконано успішно!");
    }

    function disableAction(){
        deactivate.textContent = 'Захист активовано!';
        action.textContent = 'Дія заблокована';
        action.removeEventListener('click', executeAction);
    }
    action.addEventListener('click', executeAction);
    deactivate.addEventListener('click', disableAction);
}

📝 Завдання №32.5 (Система підтвердження):
Уяви, що ми робимо форму відправки даних. Користувач натискає «Надіслати звіт». 
Система має зафіксувати клік, заблокувати кнопку, щоб уникнути повторного надсилання, і очистити пам'ять від слухача.
Крок 1. Додай у свій index.html кнопку:
<button id="submit-report-btn">Надіслати звіт</button>

Крок 2. У файлі app.js реалізуй таку логіку:
Знайди кнопку за її ID (#submit-report-btn).
Створи функцію sendReport. Всередині неї:
Виведи в консоль повідомлення: "Звіт успішно відправлено в базу даних!".
Зміни текст цієї кнопки на: "Звіт надіслано".
Заблокуй кнопку візуально, додавши їй властивість: this.disabled = true;.
Видали слухач подій click для цієї функції, щоб повністю звільнити пам'ять.
Прив'яжи функцію до кнопки через addEventListener.

✅ Рішення:
/** @type {HTMLElement} */
const btn = document.querySelector('#submit-report-btn');
if(btn){
    function sendReport(){
        console.log("Звіт успішно відправлено в базу даних!");
        btn.textContent = "Звіт надіслано";
        this.disabled = true;
        btn.removeEventListener('click', sendReport);
    }
    btn.addEventListener('click', sendReport);
}
