Тема №31: Об'єкт події (Event Object) 📦
Браузер завжди передає цей об'єкт найпершим аргументом у нашу функцію. Зазвичай розробники приймають його у змінну з назвою e або event:
button.addEventListener('click', function(e) {
    console.log(e); // Виведе величезний об'єкт із деталями події
});

У цьому об'єкті є безліч корисних властивостей:
🎯 e.target — це посилання на конкретний елемент, на якому відбулася подія (дуже схоже на this, але працює навіть у стрілкових функціях).
📄 e.target.value — динамічний вміст (текст), який знаходиться всередині елемента input або textarea в момент введення.
⌨️ e.key — якщо це подія клавіатури ('keydown'), тут зберігається назва натиснутої клавіші (наприклад, 'Enter', 'Escape', або літери).
Це відкриває можливість робити динамічний інтерфейс, який реагує на введення користувача.

⌨️ Робота з клавіатурою через e.key
Коли ми працюємо з подіями клавіатури, наприклад 'keydown' (натискання клавіші), об'єкт події автоматично містить властивість e.key. 
Вона зберігає текстову назву кнопки, яку щойно натиснув користувач.

Наприклад:
Якщо натиснути Enter, e.key буде дорівнювати "Enter".
Якщо натиснути Escape, e.key буде дорівнювати "Escape".
Якщо натиснути пробіл, e.key буде дорівнювати " ".
Це дозволяє нам створювати гарячі клавіші або керувати інтерфейсом за допомогою клавіатури, як у справжніх десктопних програмах на C#.
// Приклад відловлювання клавіші Enter
input.addEventListener('keydown', function(e) {
    if (e.key === 'Enter') {
        console.log('Користувач натиснув Enter!');
    }
});

Коли користувач натискає гарячі клавіші, нам часто потрібно знати, чи були в цей момент затиснуті спеціальні модифікатори, як-от Ctrl або Shift 
(наприклад, для комбінацій на кшталт Ctrl + Enter для надсилання повідомлення, як у багатьох месенджерах).
В об'єкті події e для цього є готові булеві властивості:
🎛️ e.ctrlKey — повертає true, якщо під час події був затиснутий Ctrl.
⬆️ e.shiftKey — повертає true, якщо був затиснутий Shift.

🛑 Скасування стандартної поведінки (e.preventDefault())
У браузері є HTML-елементи, які мають свою «вбудовану» поведінку за замовчуванням. Наприклад:
Коли ти клікаєш на посилання <a href="...">, браузер автоматично перезавантажує сторінку і переходить за адресою.
Коли ти натискаєш кнопку всередині форми <form>, браузер намагається відправити дані на сервер і знову ж таки перезавантажує сторінку.
У сучасних JS-додатках (як і у C# Web API / Blazor) ми часто хочемо обробляти такі події самостійно через JavaScript, 
без перезавантаження сторінки. Для цього в об'єкта події e є спеціальний метод — e.preventDefault().

Приклад використання:
const link = document.querySelector('a');

link.addEventListener('click', function(e) {
    e.preventDefault(); // Зупиняємо перехід за посиланням!
    console.log("Посилання клікнули, але нікуди не переходимо.");
});

            status.textContent = 'Швидкість зменшено!';
            status.style.color = 'orange';
