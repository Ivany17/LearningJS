Тема №30: Обробка подій (Event Listeners) 🌐

🌐 Що таке подія (Event)?
Подія — це сигнал від браузера про те, що на сторінці щось відбулося.
Користувач може клікнути мишкою, ввести текст у поле або прокрутити сторінку.
Усі ці дії браузер фіксує як події. Наше завдання — навчити JavaScript реагувати на них.
Для C#-розробників це працює точно так само, як механізм подій (Events).

🔌 Що таке обробник подій (Event Listener)?
Event Listener (слухач подій) — це спеціальна функція, яку ми "прикріплюємо" до HTML-елемента.
Вона сидить і "слухає" елемент, очікуючи на конкретну дію (наприклад, клік).
Щойно подія відбувається, слухач миттєво запускає наш JS-код.

📥 Метод .addEventListener()
Щоб додати слухача, в JavaScript використовується метод addEventListener().
Він приймає два головні параметри:
Назву події у вигляді рядка (наприклад, 'click' — для кліку мишкою).
Функцію, яка має виконатися, коли ця подія відбудеться.

Приклад синтаксису:
button.addEventListener('click', () => {
    // Код, який спрацює ПІСЛЯ кліку
});

📖 Словник базових подій JavaScript
Кожне слово-подія — це суворий системний рядок, який JavaScript розпізнає автоматично. Ми маємо писати їх у лапках без помилок:
🖱️ 'click' — подія відбувається, коли користувач один раз клікає лівою кнопкою миші на елемент.
🖱️ 'dblclick' — подія подвійного кліку (double click).
📥 'mouseenter' — подія спрацьовує, коли курсор миші просто наводиться на елемент (входить у його межі).
📤 'mouseleave' — подія спрацьовує, коли курсор миші залишає межі елемента.
⌨️ 'keydown' — подія виникає, коли користувач натискає будь-яку клавішу на клавіатурі.
📝 'input' — подія виникає в той момент, коли користувач вводить або змінює будь-який символ всередині текстового поля.

🔑 Ключове слово this в обробниках подій
Коли всередині .addEventListener() ми використовуємо звичайну функцію, JavaScript автоматично записує в службове слово this той самий HTML-елемент, 
на який ми клікнули або навели мишу.
Для C#-розробника це дуже знайома логіка: this вказує на поточний об'єкт, який викликав подію.
⚠️ Важливий нюанс JS: Щоб слово this всередині слухача подій вказувало саме на наш HTML-елемент, 
ми маємо замість стрілкової функції () => { ... } використати класичну функцію function() { ... }.
Подивіться, як це спрощує код. Замість того, щоб знову писати ім'я змінної (button.style...), ми пишемо this.style...:
button.addEventListener('click', function() {
    this.style.backgroundColor = 'red'; // this — це і є наша кнопка!
});


📝 Завдання №30.1 (Перший клік):
Давай створимо максимально просту структуру, щоб побачити обробку подій у дії.
Крок 1. Заміни код у своєму index.html на цей мінімальний варіант з однією кнопкою:

<!DOCTYPE html>
<html lang="uk">
<head>
    <meta charset="UTF-8">
    <title>Перша подія</title>
</head>
<body>

    <button id="trigger-btn">Натисни мене</button>

    <script src="app.js"></script>
</body>
</html>

Крок 2. Очисть файл app.js і напиши код за такими підказками:
За допомогою document.querySelector знайди кнопку за її ID #trigger-btn (не забудь додати універсальний коментар / @type {HTMLElement} */).
Перевір кнопку через if.
Всередині блоку if додай до кнопки слухач подій addEventListener для події 'click'.
Всередині слухача подій напиши console.log("Кнопку успішно натиснуто!");.
Після цього відкрий сторінку в браузері, натисни кнопку та перевір вкладку Console (F12).

✅ Рішення:
/** @type {HTMLElement} */
const button = document.querySelector('#trigger-btn');
if(button){
    button.addEventListener('click', () => {console.log("Кнопку успішно натиснуто!")});
}

📝 Завдання №30.2 (Зміна типу події):
Перепиши код у своєму app.js так, щоб ми задіяли інші події:
Знайди кнопку за її ID #trigger-btn (з коментарем типу та перевіркою if).
Замість одного слухача 'click', додай до цієї ж кнопки два окремі слухачі подій:
Перший слухач має реагувати на подію 'mouseenter' (наведення миші). 
Всередині нього напиши код, який фарбує фон кнопки в жовтий колір: button.style.backgroundColor = 'yellow';.
Другий слухач має реагувати на подію 'mouseleave' (відведення миші). 
Всередині нього напиши код, який повертає кнопці білий або звичайний фон: button.style.backgroundColor = 'white';.
Спробуй зберегти код, оновити сторінку і просто поводити мишкою туди-сюди по кнопці, не клікаючи на неї.

✅ Рішення:
/** @type {HTMLElement} */
const button = document.querySelector('#trigger-btn');
if(button){
    button.addEventListener('click', () => {console.log("Кнопку успішно натиснуто!")});
    button.addEventListener('mouseenter', () => {button.style.backgroundColor = 'yellow'});
    button.addEventListener('mouseleave', () => {button.style.backgroundColor = 'white'});
}

📝 Завдання №30.3 (Динамічний текст кнопки):
Очисть файл app.js і напиши код, який змінюватиме і колір, і текст кнопки за допомогою this:
Знайди кнопку за її ID #trigger-btn (з коментарем типу та перевіркою if).
Додай до кнопки слухач подій 'click'. Обов'язково використай класичну функцію: function() { ... }.
Всередині функції за допомогою this зроби дві дії:
Зміни текст кнопки на: "Мене натиснуто!" (через this.textContent).
Зміни колір тексту кнопки на зелений ('green') або будь-який інший колір на твій вибір (через this.style.color).
Збережи код, онови сторінку в браузері та натисни на кнопку, щоб перевірити, як вона сама себе оновить.

✅ Рішення:
/** @type {HTMLElement} */
const button = document.querySelector('#trigger-btn');
if(button){
    button.addEventListener('click', function(){
        this.textContent = "Мене натиснуто!";
        this.style.color = 'green';
    });
}
