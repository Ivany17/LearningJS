Тема №5: Оператори порівняння та Логіка ⚖️
У JavaScript є два типи порівняння: суворе та несуворе. Це одна з найважливіших тем, де розробники часто припускаються помилок.

1. Стандартні оператори порівняння
Тут усе як у математиці:
> — більше
< — менше
>= — більше або дорівнює
<= — менше або дорівнює

2. Дві великі різниці: == проти ===
Ось тут JavaScript показує свій характер:
== (Несуворе порівняння): JS намагається бути "корисним" і приводить типи до одного знаменника перед порівнянням.
5 == "5" поверне true (бо JS перетворив рядок на число).
=== (Суворе порівняння): Перевіряє і значення, і тип. Це золотий стандарт у розробці.
5 === "5" поверне false (бо число не є рядком).

Завжди використовуй === (суворе порівняння). 
Це вбереже тебе від дивних помилок, коли програма випадково вважає число 0 порожнім рядком або значенням false.

3. Логічні оператори (AND, OR, NOT)
Вони допомагають створювати складні умови:
&& (І / AND): Повертає true, тільки якщо всі умови істинні.
|| (АБО / OR): Повертає true, якщо хоча б одна умова істинна.
! (НІ / NOT): Перевертає значення (робить true із false і навпаки).

📝 Завдання №5 (Comparison):
Давай перевіримо, як ці оператори працюють у твоєму коді.
Створи дві змінні: minAge = 18 та userAge = 20.
Виведи в консоль результат порівняння userAge > minAge.
Порівняння типів: Виведи в консоль результат 10 == "10" та 10 === "10".
Логічна задача: Створи змінну hasTicket = true та isAdult = true. Виведи результат умови: "Людина може зайти, якщо вона доросла ТА має квиток".
Додатковий тест: Зміни hasTicket на false і подивись, як зміниться результат у консолі.
Логічне "АБО": Спробуй використати оператор || (OR) між цими ж змінними. Що він виведе, якщо одна з них true, а інша false?

✅ Рішення:
let minAge = 18;
let userAge = 20;
console.log(userAge > minAge); // true
console.log(10 == "10"); // true
console.log(10 === "10"); // false

let hasTicket = true;
let isAdult = true;
console.log(hasTicket && isAdult); // true
hasTicket = false;
console.log(hasTicket && isAdult); // false
console.log(hasTicket || isAdult); // true
