Тема №12: Стрілочні функції (Arrow Functions) 🏹
➡️ Стрілочні функції — це компактний синтаксис написання функцій, введений у 2015 році (ES6). Вони є стандартом для сучасної розробки на JavaScript.
У JavaScript стрілочна функція технічно є виразом (expression). Це означає, що ти створюєш "анонімну" функцію і кладеш її у змінну, як звичайне число або об'єкт.
Оскільки ми хочемо, щоб це посилання на функцію залишалося постійним протягом усієї роботи програми, const (скорочення від constant) підходить для цього ідеально.

1. Базовий синтаксис
Замість ключового слова function, ми використовуємо "стрілку" => після дужок з параметрами.

// Класична функція
function sayHi() {
  console.log("Привіт!");
}

// Стрілочна функція
const sayHi = () => {
  console.log("Привіт!");
};
// Якщо код займає один рядок, return пишеться автоматично, але якщо більше - return треба писати вручну, тобто
Немає {} — return підставляється автоматично (тільки для одного рядка).
Є {} — return потрібно писати власноруч.
console.log — це лише дзеркало, воно показує, що відбувається, але не зберігає дані.
return — це поштовий голуб, який приносить результат виконання функції в те місце, де її викликали.

2. У стрілочних функцій є три правила "мінімалізму":
Без дужок для параметра: Якщо параметр лише один, круглі дужки () можна не ставити.
const double = n => n * 2;

Без return та {}: Якщо код займає один рядок, фігурні дужки та слово return прибираються.
const sum = (a, b) => a + b;

З дужками: Якщо параметрів нуль або два і більше, круглі дужки обов'язкові.
const greet = () => "Hello!";

⚠️ Важливо: Якщо ти використовуєш фігурні дужки { }, то JavaScript автоматично не поверне значення. У такому разі слово return писати обов'язково.

3. Головна пастка: Hoisting (Підняття) ⏫
⬆️ Звичайні функції (function): Піднімаються вгору. Їх можна викликати до того, як вони оголошені в коді.
⬇️ Стрілочні функції: Зберігаються у змінних (const або let), тому не піднімаються.

Порядок має значення:
// ❌ Помилка: ReferenceError
sayHello(); 
const sayHello = () => console.log("Hi!");

// ✅ Правильно
const sayHello = () => console.log("Hi!");
sayHello();

4. Коли що використовувати?
Стрілочні функції: Для коротких операцій, математики, обробки списків та всюди, де потрібен чистий і короткий код.
Звичайні функції: Якщо тобі потрібне "підняття" (викликати раніше, ніж оголосити) або при роботі зі складними об'єктами.

5. Стрілочні функції та тернарний оператор ⚡
Коли ми маємо просту умову if/else, програмісти часто поєднують стрілочні функції з тернарним оператором. 
Це дозволяє знову повернутися до запису в один рядок.

❇️ Синтаксис тернарного оператора:
умова ? значення_якщо_true : значення_якщо_false

📝 Завдання №12 (Сучасний підхід):
Створи стрілочну функцію convertCelsiusToFahrenheit.
Вона повинна приймати один параметр — celsius.
Формула: F = C * 1.8 + 32
Використай скорочений запис (в один рядок без фігурних дужок), щоб функція одразу повертала результат.
Виведи в консоль результат для 25°C.

✅ Рішення:
let celsius = 25;

const convertCelsiusToFahrenheit = (celsius) => celsius * 1.8 + 32;
let fahrenheit = convertCelsiusToFahrenheit(celsius);
console.log(`${celsius}°C перетворилися на ${fahrenheit}°F`);

📝 Завдання №12.2 (Закріплення автоматизму):
Напиши три стрілочні функції в один рядок (максимально скорочено):
mult — приймає a та b, повертає їхній добуток.
isPositive — приймає одне число n, повертає true, якщо воно більше 0, і false, якщо менше або дорівнює. (Підказка: просто n > 0).
repeatText — приймає рядок text, повертає цей же рядок, написаний двічі (через плюс або шаблонний рядок).

✅ Рішення:
const mult = (a, b) => a * b;
const isPositive = n => n > 0;
const repeatText = (string) => string + string;

console.log(mult(3, 7));
console.log(isPositive(14));
console.log(repeatText("text"));

📝 Завдання №12.3 (Логіка всередині стрілки):
Давай спробуємо написати стрілочну функцію, яка трохи складніша за один рядок. 
Якщо код займає більше одного рядка, нам потрібні фігурні дужки {} та слово return.
Створи стрілочну функцію checkWeather.
Параметр: temp (температура).
Всередині (у фігурних дужках):
Якщо temp більше 25 — поверни рядок "Спекотно ☀️".
В іншому випадку — поверни "Комфортно ☁️".
Виклич її для температури 30 та 15 і виведи результати.

✅ Рішення:
const checkWeather = (temp) => {
    if(temp > 25){
        return `Спекотно ☀️`;
    } else {
        return `Комфортно ☁️`;
    }
};

console.log(checkWeather(30));
console.log(checkWeather(15));

📝 Завдання №12.4 (Фінальне тренування на стрілки):
Давай напишемо функцію для перевірки доступу, використовуючи цей супер-короткий стиль.
Створи стрілочную функцію checkAccess.
Вона приймає один параметр — userRole.
Використай тернарний оператор, щоб:
Якщо userRole дорівнює "admin", функція повертала "Доступ дозволено ✅".
В усіх інших випадках — "Доступ заборонено ❌".
Напиши це в один рядок (без фігурних дужок та без return).
Перевір її роботу, викликавши з аргументом "admin", а потім з аргументом "guest".

✅ Рішення:
const checkAccess = userRole => userRole === "admin" ? "Доступ дозволено ✅" : "Доступ заборонено ❌";
console.log(checkAccess("admin"));
console.log(checkAccess("guest"));
