Тема №11: Функції (Functions) — Основи 📦
Функція — це іменований блок коду, який можна викликати (запускати) скільки завгодно разів у будь-якому місці програми.

1. Навіщо вони потрібні? 🤔
Уяви, що тобі треба 10 разів у різних частинах коду виводити гарне привітання. 
Замість того, щоб копіювати console.log щоразу, ти один раз створюєш функцію і просто кличеш її на ім'я.

🚫 DRY (Don't Repeat Yourself) — не повторюйся. Це золоте правило програміста.

2. Оголошення функції (Declaration) 🏗️
Для створення функції використовується ключове слово function, потім ім'я, круглі дужки () та фігурні дужки {}.

function sayHello() {
  console.log("Привіт! Я — твоя перша функція. 👋");
}

3. Виклик функції (Call) 📞
Сама по собі функція не запуститься. Її треба "покликати", написавши ім'я зі скобками:

sayHello(); // Викличе функцію і виведе текст у консоль

4. Ми можемо передавати у функцію дані.
Параметри — це змінні, які ми вказуємо в дужках при створенні функції (як порожні коробки для даних).
Аргументи — це реальні значення, які ми кладемо в ці коробки при виклику.

function greetUser(name) { // name — це параметр
  console.log(`Привіт, ${name}! Ласкаво просимо.`);
}

greetUser("Іван"); // "Іван" — це аргумент
greetUser("Олексій");

5. В реальному житті нам часто потрібно, щоб функція обчислила щось і повернула нам цей результат, 
аби ми могли працювати з ним далі (наприклад, додати до іншого числа або зберегти в базу даних).
❗️ return — це вихід із функції. Він каже: "Я закінчила роботу, ось твоє значення".

function getSum(a, b) {
  return a + b; // Функція повертає суму
}

let result = getSum(5, 5); // Зберігаємо повернуте значення у змінну
console.log(result + 10); // Тепер ми можемо далі працювати з цим числом (виведе 20)

⚠️ Важливо: Як тільки JS зустрічає return, він негайно виходить із функції. Будь-який код, 
написаний після return всередині тієї ж функції, ніколи не виконається.

📝 Завдання №11 (Перша функція):
Давай почнемо з простого, але важливого кроку.
Створи функцію з ім'ям showWarning.
Всередині функції напиши три команди console.log:
"УВАГА!"
"Система зафіксувала помилку!"
"Будь ласка, перевірте з'єднання."
Виклич цю функцію три рази поспіль.
Подивись у термінал — ти побачиш 9 рядків тексту, хоча в самому коді (поза функцією) ти написав лише 3 рядки виклику.

✅ Рішення:
showWarning();
showWarning();
showWarning();

function showWarning() {
    console.log("УВАГА!");
    console.log("Система зафіксувала помилку!");
    console.log("Будь ласка, перевірте з'єднання.");
}

📝 Завдання №11.2 (Персоналізація):
Давай зробимо функцію, яка буде виводити результат математичної операції для будь-яких чисел.
Створи функцію multiply.
Додай їй два параметри: a та b.
Всередині функції виведи в консоль повідомлення: "Результат множення: [тут має бути результат множення a на b]".
Виклич функцію тричі з різними парами чисел (аргументами). Наприклад: multiply(5, 10), multiply(2, 8) тощо.

✅ Рішення:
multiply(5, 10);
multiply(2, 8);
multiply(17, 6);

function multiply(a, b) {
    console.log(`Результат множення: ${a} * ${b} = ${a * b}`);
}

📝 Завдання №11.3 (Повернення значення):
Давай перетворимо твій калькулятор множення на більш корисний інструмент.
Створи функцію calculateArea.
Додай їй два параметри: width (ширина) та height (висота).
Замість console.log, використай return, щоб повернути результат множення ширини на висоту.
Поза функцією:
Виклич функцію для прямокутника 5x10 і збережи результат у змінну roomArea.
Виклич функцію для прямокутника 3x4 і збережи результат у змінну kitchenArea.
Виведи в консоль загальну площу, просто додавши ці дві змінні: console.log(roomArea + kitchenArea).

✅ Рішення:
let roomArea = calculateArea(5, 10);
let kitchenArea = calculateArea(3, 4);

function calculateArea(width, height) {
    return width * height;
}

console.log(`Загальна площа квартири: ${roomArea + kitchenArea} м2.`)
