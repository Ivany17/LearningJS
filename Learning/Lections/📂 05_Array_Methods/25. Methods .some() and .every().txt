Тема №25: Методи .some() та .every() ✅

🚀 Ці два методи використовуються для швидкої перевірки масиву на відповідність певній умові. 
Вони не змінюють масив і не повертають нову колекцію, а лише одне логічне значення: true або false.

🔹 Метод .some()
Цей метод перевіряє, чи є в масиві хоча б один елемент, що задовольняє умову. 
Це працює як логічне "АБО" (||) для всієї колекції. 
Як тільки метод знаходить перший елемент, який підходить під умову, він негайно повертає true і припиняє перебір. 
Якщо жоден елемент не підійшов — повертає false.
Аналог у C# LINQ: .Any().

🔹 Метод .every()
Цей метод перевіряє, чи кожен елемент масиву відповідає умові. 
Це працює як логічне "І" (&&). Якщо він зустріне хоча б один елемент, який не відповідає умові, він негайно повертає false. 
true повернеться лише тоді, коли всі елементи без винятку пройшли перевірку.
Аналог у C# LINQ: .All().

💡 Важливий нюанс: Обидва методи приймають callback-функцію, яка отримує поточний елемент масиву (традиційно позначаємо як n або el).

Приклад використання:
const nums = [1, 5, 10];
nums.some(n => n > 8); // true (бо 10 > 8)
nums.every(n => n > 8); // false (бо 1 та 5 не більше 8)

📝 Завдання №25.1 (Перевірка статусів замовлень):
Створи масив об'єктів orders із трьома елементами. Кожен об'єкт має id та булеву властивість isReady. 
Нехай у двох замовлень буде true, а у одного — false.
Використай метод .some(), щоб перевірити, чи є в масиві замовлення, які не готові (isReady === false). 
Виведи результат (true/false) у консоль з пояснювальним текстом.
Використай метод .every(), щоб перевірити, чи всі замовлення в масиві готові (isReady === true). Виведи результат у консоль.

✅ Рішення:
const orders = [
    { id: 1, isReady: true },
    { id: 2, isReady: false },
    { id: 3, isReady: true },
];

const some = orders.some(o => o.isReady === false);
const every = orders.every(o => o.isReady);

console.log(`Чи є в масиві замовлення, які ще не готові? ${some}`);
console.log(`Чи всі замовлення в масиві готові? ${every}`);

📝 Завдання №25.2: Валідація паролів та доступів
Уяви, що ти перевіряєш список паролів користувачів на безпеку та наявність адмін-прав у групі.
Створи масив, де будуть name, password і role (admin, або user).
Твоя задача:
Використай .some(), щоб перевірити, чи є серед користувачів хоча б один з роллю 'admin'.
Використай .every(), щоб перевірити, чи всі паролі в масиві мають довжину більше 5 символів (.length > 5).
Виведи обидва результати у консоль з коротким текстом-поясненням.

✅ Рішення:
const users = [
    { name: "Ivan", password: "123", role: "admin" },
    { name: "Oleg", password: "securePassword77", role: "user" },
    { name: "Anna", password: "qwerty12345", role: "user" }
];

const some = users.some(u => u.role == "admin");
const every = users.every(u => u.password.length > 5);

console.log(`Чи є серед користувачів 'admin'? ${some}`);
console.log(`Чи довжина паролей всіх користувачів більше за 5 символів? ${every}`);
