Тема №28: Ланцюжки методів 🔗

🚀 Ланцюжки методів — це одна з найпотужніших можливостей JavaScript, 
яка дозволяє викликати кілька методів масиву один за одним у межах одного виразу. 
Це робить код компактним, читабельним і дозволяє будувати справжні «конвеєри» для обробки даних.

🔹 Як це працює?
Більшість методів масивів, які ми вивчили (.filter(), .map(), .sort(), .flat()), повертають новий масив. 
Оскільки результатом виконання методу є масив, ми можемо одразу (через крапку) викликати наступний метод масиву.

🔹 Головні переваги:
Відсутність зайвих змінних: Тобі не потрібно створювати filteredArray, потім mappedArray тощо.
Логічна послідовність: Ти чітко бачиш шлях трансформації даних: Спочатку відфільтрували ➡️ Потім змінили форму ➡️ Потім відсортували.

⚠️ Важливо: Методи, які повертають не масив (наприклад, .reduce(), .find(), .every(), .some()), зазвичай ставляться в самому кінці ланцюжка, 
бо після них метод масиву викликати вже не вийде (адже вони повернули число, об'єкт або boolean).

Приклад структури:
const result = array
  .filter(item => item.active) // Повертає масив
  .map(item => item.name)      // Повертає масив
  .sort();                     // Повертає масив

📝 Завдання №28.1 (Конвеєр обробки замовлень):
Уяви, що ти працюєш над інтерфейсом магазину. Тобі потрібно отримати назви дорогих товарів у алфавітному порядку.
Створи масив об'єктів products:
{ title: "Smartphone", price: 500 }
{ title: "Case", price: 20 }
{ title: "Tablet", price: 300 }
{ title: "Cable", price: 15 }
Використай ланцюжок методів, щоб виконати наступне в одному виразі:
    .filter(): залишити лише товари, ціна яких більша за 100.
    .map(): отримати масив, що містить лише назви (title) цих товарів.
    .sort(): відсортувати отримані назви за алфавітом.
Виведи фінальний результат у консоль.

✅ Рішення:
const products = [
    { title: "Smartphone", price: 500 },
    { title: "Case", price: 20 },
    { title: "Tablet", price: 300 },
    { title: "Cable", price: 15 },
];

const chain = products.filter(p => p.price)
                        .map(p => p.title)
                        .sort((a, b) => a.localeCompare(b));
console.log(chain);

📝 Завдання №28.2 (Аналітика студентів):
Уяви, що в тебе є дані про студентів та їхні оцінки. 
Тобі потрібно отримати список імен "відмінників", відсортований за алфавітом, але дані приходять у дещо "брудному" вигляді.
Створи масив data, в якому будуть name і масив з трьох оцінок grades.
Використай ланцюжок методів, щоб:
    .filter(): залишити тільки тих, у кого середній бал більший за 90.
(Підказка: для розрахунку середнього балу всередині filter використай .reduce() для масиву grades і поділи на його довжину).
    .map(): перетворити об'єкт студента на рядок, де ім'я починається з великої літери.
(Підказка: n.name[0].toUpperCase() + n.name.slice(1)).
    .sort(): відсортувати імена за алфавітом.
Виведи результат у консоль.

✅ Рішення:
const data = [
    { name: "ivan", grades: [80, 90, 100] },
    { name: "anna", grades: [95, 98, 92] },
    { name: "oleg", grades: [60, 70, 65] },
    { name: "dmitro", grades: [100, 100, 98] }
];

const excellentStudents = data.filter(d => (d.grades.reduce((acc, n) => acc + n, 0) / d.grades.length) > 90)
                                .map(d => d.name[0].toUpperCase() + d.name.slice(1))
                                .sort((a, b) => a.localeCompare(b));
console.log(excellentStudents);

📝 Завдання №28.3 (Система фільтрації замовлень):
У тебе є масив замовлень, де деякі суми вказані як рядки (так іноді приходить з API), а деякі замовлення ще не оплачені.
Створи масив orders, де будуть id, amount, status (paid, pending).
Використай ланцюжок методів, щоб:
    .filter(): залишити тільки ті замовлення, де статус — "paid".
    .map(): перетворити кожен об'єкт так, аби поле amount стало числом (Number або +), бо зараз це рядок. Поверни масив тільки цих чисел.
    .filter(): (так, ще раз!) залиш тільки ті суми, що більші за 130.
    .reduce(): порахуй загальну суму цих замовлень.
Виведи фінальну суму в консоль.

✅ Рішення:
const orders = [
    { id: 1, amount: "150", status: "paid" },
    { id: 2, amount: "400", status: "pending" },
    { id: 3, amount: "120", status: "paid" },
    { id: 4, amount: "200", status: "paid" },
    { id: 5, amount: "50", status: "pending" }
];

const paidOrders = orders.filter(o => o.status == "paid")
                        .map(o => Number(o.amount))
                        .filter(o => o > 130)
                        .reduce((acc, n) => acc + n, 0);
console.log(paidOrders);

📝 Завдання №28.4 (Фінальний бос ланцюжків):
Давай поєднаємо все, що ми вивчили, включаючи .flat() та .includes(), в один великий логічний ланцюжок.
Створи масив warehouse, де будуть category і масив items.
Використай ланцюжок методів, щоб:
    .map(): дістати з об'єктів лише масиви items.
    .flat(): перетворити масив масивів на один плоский масив назв.
    .map(): прибрати зайві пробіли на початку та в кінці кожного слова (використай метод рядків .trim()).
    .filter(): залишити тільки ті назви, які НЕ включають (не .includes()) слово "PC".
    .sort(): відсортувати результат за алфавітом.
Виведи фінальний масив у консоль.

✅ Рішення:
const warehouse = [
    { category: "Electronics", items: [" Laptop ", "Smartphone", " PC "] },
    { category: "Accessories", items: ["  Mouse", "Keyboard ", " Monitor"] }
];

const filter = warehouse.map(w => w.items)
                        .flat()
                        .map(w => w.trim())
                        .filter(w => !w.includes('PC'))
                        .sort((a, b) => a.localeCompare(b));
console.log(filter);
