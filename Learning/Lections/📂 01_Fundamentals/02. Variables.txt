ема №2: Змінні в JavaScript — Скрині для даних (Variables) 📦
В C# ти звик до суворої типізації: int age = 25; або string name = "Ivan";. JavaScript у цьому плані набагато ліберальніший.
У сучасному JS є два основні способи створити змінну:

let — використовується для змінних, значення яких може змінюватися в ході програми. (Аналог звичайних змінних у C#).
const — використовується для значень, які ніколи не змінюються (константи). Якщо ти спробуєш змінити const після створення, програма "впаде" з помилкою.
⚠️ Важливо: Ключове слово (let або const) пишеться лише один раз при створенні змінної. Щоб оновити значення, ми просто пишемо ім'я змінної.
👎 var (застарілий) — так створювали змінні до 2015 року. Він дуже схожий на let, але має «погані манери». Наприклад, var дозволяє тобі 
оголосити змінну з тим самим ім'ям кілька разів 
(щойно ти бачив, як let видав помилку в такій ситуації, а var би промовчав і просто все переплутав).
Порада: У сучасному коді ми не використовуємо var, але маємо знати про його існування.

Головна відмінність: Тобі не потрібно писати int чи string. JavaScript сам зрозуміє, що лежить у "скрині", коли ти покладеш туди значення.

Приклад:
let age = 25; — JS сам зрозумів, що це число.
let name = "Іван"; — JS сам зрозумів, що це рядок.

📝 Завдання №2:
Давай попрацюємо зі змінними. 
Створи змінну для назви твого міста (подумай, чи буде вона змінюватися в ході програми, і обери let або const).
Створи змінну для твого віку.
Виведи обидві ці змінні в консоль.
Нижче в коді зміни значення змінної з віком (наприклад, вік став на 1 рік більшим) і виведи її в консоль ще раз.
Запусти файл командою node app.js (або іншою назвою, якщо ти перейменував файл).

✅ Рішення:
const city = "Kyiv";
let age = 27;
console.log(city);
console.log(age);

age = 28;
console.log(age);

📝 Завдання №2.1 (Variables Recap):
Це фінальне завдання, щоб закріпити різницю між властивостями:
Спробуй оголосити змінну через var з назвою test і дати їй значення "Перший".
Рядком нижче знову напиши var test = "Другий";.
Виведи test у консоль і подивися, чи видасть JS помилку (як це було з let).
Нижче створи const з назвою myCountry. Спробуй змінити її значення на іншу країну.
Запусти код і подивися на результат у терміналі.

✅ Рішення:
var test = "Перший";
var test = "Другий"; // програма продовжить працювати, навіть коли властивість змінено

console.log(test); // Другий

const myCountry = "Ukraine";
const myCountry = "Italy"; // програма зламається, оскільки const не терпить змін своїх властивостей
